# AgentLens 架构说明

## 整体架构

AgentLens 是为 LangChain / LangGraph Agent 设计的轻量可观测性平台。三部分解耦，通过 HTTP + JSON 通信。

```
┌─────────────────────┐         HTTP/JSON         ┌──────────────────────┐
│  AI Agent 项目      │  ─── POST /api/v1/traces ─▶ │  AgentLens Server    │
│                     │                            │  (FastAPI)           │
│  ┌──────────────┐   │                            │                      │
│  │ agent.invoke │   │                            │  ┌────────────────┐  │
│  │     +        │   │                            │  │  API Router    │  │
│  │  TraceCB     │   │                            │  └────────┬───────┘  │
│  │  (langchain_ │   │                            │           │          │
│  │  trace_cb)   │   │                            │  ┌────────▼───────┐  │
│  └──────────────┘   │                            │  │  Service       │  │
│                     │                            │  └────────┬───────┘  │
└─────────────────────┘                            │           │          │
                                                   │  ┌────────▼───────┐  │
                                                   │  │  Repository    │  │
                                                   │  │  (memory/file/ │  │
                                                   │  │   pluggable)   │  │
                                                   │  └────────────────┘  │
                                                   └──────────┬───────────┘
                                                              │
                                                              │ HTTP
                                                              ▼
                                                   ┌──────────────────────┐
                                                   │  AgentLens UI        │
                                                   │  (React + Vite)      │
                                                   │                      │
                                                   │  - Trace 列表        │
                                                   │  - 多视图详情        │
                                                   │  - 会话分组          │
                                                   └──────────────────────┘
```

## 组件职责

### 1. Callback 客户端（agentlens-cb，import `agentlens_cb`）

- 嵌入 AI Agent 项目，作为 LangChain `BaseCallbackHandler` 注册
- 监听 `on_chain_*` / `on_llm_*` / `on_tool_*` / `on_chat_model_*` 等事件
- 启发式识别 middleware（在 `on_chain_start` 注入 `is_middleware` / `middleware_name` / `node_name`）
- 提取 `thread_id`（从 `metadata.configurable.thread_id`）
- 根 chain 结束时一次性 POST 到后端
- 守护线程异步发送，进程退出时通过 `atexit` 等待完成
- 失败不抛异常：短超时 + 熔断冷却 30s

依赖：`langchain-core>=0.3.0` + `httpx>=0.27.0`，无其他依赖。

### 2. Server（agentlens-server）

FastAPI + Pydantic v2。

- **API**：`/api/v1/traces` 提供 `POST`（接收 trace）/ `GET`（列表/详情）/ `GET /by_thread/{id}`（按 thread 合并）
- **存储**：抽象 `TraceRepository` 接口
  - `MemoryTraceRepository`：进程内 dict（默认，重启丢失）
  - `FileTraceRepository`：每 trace 一个 JSON 文件（轻量持久化）
  - 扩展：继承 `TraceRepository` 并在 `app/dependencies.py` 注册
- **CORS**：通过 `TRACE_CORS_ORIGINS` 配置允许的前端来源
- **配置**：`pydantic_settings` + `.env`，`TRACE_*` 前缀

### 3. UI（agentlens-ui）

React 18 + Vite + Tailwind + TanStack Query。

- **列表页**（`/`）：按 `thread_id` 分组，每组显示若干 trace
- **Trace 详情页**（`/traces/:traceId`）：4 个 tab
  - 调用树（缩进 + 节点类型颜色 + middleware 高亮 + IO 展开）
  - 时间线（按时间排序的扁平事件 + 来源标号）
  - 消息流（按 type 分类，summary 紫色横幅，middleware 橙色边界）
  - IO（所有事件 JSON 索引 + 详情）
- **Thread 详情页**（`/threads/:threadId`）：合并该 thread 全部 trace
  - 调用树 / 时间线 / 消息流（跨 trace 合并展示）
  - 每 trace 用「调用 #N」分界条分割

## 数据模型

### Event（callback 推送的最小单元）

```json
{
  "event_id": "uuid",
  "run_id": "uuid",
  "parent_run_id": "uuid|null",
  "event_type": "chain_start | chain_end | llm_start | llm_end | tool_start | tool_end | chat_model_start | retriever_start | retriever_end | agent_action | agent_finish | text | *_*_error",
  "timestamp": "ISO8601",
  "name": "...",
  "serialized": { "id": ["langchain", "Module", "Class"], "name": "..." },
  "inputs": <任意 JSON>,
  "outputs": <任意 JSON>,
  "metadata": { "langgraph_node": "agent", "configurable": { "thread_id": "..." } },
  "tags": ["..."],
  "level": "info | error",
  "is_middleware": true,
  "middleware_name": "PIIMiddleware[custom].before_model",
  "node_name": "PIIMiddleware[custom].before_model"
}
```

### Trace

```json
{
  "trace_id": "uuid",
  "name": "weather-agent",
  "tags": ["prod"],
  "metadata": { "app": "..." },
  "thread_id": "session-1",
  "start_time": "...",
  "end_time": "...",
  "duration_ms": 1234,
  "status": "succeeded | failed | running",
  "events": [Event, ...],
  "messages": [提取自最终 chain_end 的输出],
  "structured_response": {...}
}
```

## 数据流

1. AI Agent 项目调用 `agent.invoke(input, config={"callbacks": [TraceCB], "configurable": {"thread_id": "..."}})`
2. LangChain 触发 callback 事件（on_chain_start / on_llm_end / ...）
3. callback 在内存累计 events，根 chain 结束时打包
4. 守护线程 POST 到 `http://server:8000/api/v1/traces`
5. Server 接收 → 校验 → 持久化到 Repository
6. UI 通过 React Query 拉取列表 / 详情，渲染多视图
7. 多 trace 同 thread 在 UI 层合并展示

## 关键设计决策

### 1. 客户端包命名
发布名 `agentlens-cb`（pip install 显示）+ import 名 `agentlens_cb`（Python import 路径）。
- 命名规则：pip 用连字符 `-`（PEP 508），import 用下划线 `_`（PEP 8）
- 两者指向同一个包
- 项目重命名为 AgentLens 后，import 名也同步更新（如有需要可继续兼容旧名）

### 2. 后端是只写一次的事务
- callback 在根 chain 结束时**一次性** POST 整个 trace
- 不支持增量推送、不支持流式
- 优点：实现简单，事务一致
- 限制：单 trace 大小受 HTTP 请求体限制（默认无限制，按需配置）

### 3. Middleware 识别用启发式
- 没有"显式注册"机制
- 三套策略：tag 前缀 / 类名后缀 / langgraph_node 兜底
- 优点：用户零配置
- 限制：可能误判（如果用户 chain 类名以 Middleware 结尾）

### 4. 存储可插拔
- 内存（默认）：开发 / 演示
- 文件：单实例轻量持久化
- 自定义：继承 `TraceRepository` 实现数据库 / OSS

### 5. 失败降级
callback 推送失败**不会**影响 agent 执行：
- 2s 超时
- 30s 熔断冷却
- 守护线程异步
- 进程退出时 `atexit` 等待最多 3s
