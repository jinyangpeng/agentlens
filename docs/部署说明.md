# AgentLens · 部署说明

## 方式 A：Docker Compose（推荐）

### 一键启动（生产模式）

```bash
# 启动（首次自动构建镜像）
docker compose up -d

# 等待健康检查通过
docker compose ps

# 访问
#  - 前端：    http://localhost:8080
#  - 后端：    http://localhost:8000
#  - Swagger： http://localhost:8000/docs

# 查看日志
docker compose logs -f

# 停止（保留数据卷）
docker compose down

# 停止并清除数据
docker compose down -v
```

### 自定义配置

```bash
# 复制环境变量模板
cp .env.example .env

# 修改后重新启动
# （docker compose 自动重新加载 .env）
docker compose up -d
```

可调整的变量：
- `TRACE_STORAGE_BACKEND`：`memory`（默认）/`file`（持久化）
- `BACKEND_PORT` / `FRONTEND_PORT`：对外端口
- `TRACE_CORS_ORIGINS`：CORS 允许的前端来源（默认 `localhost:5173, 8080`）

### 持久化

`agentlens-data` 卷挂载到 backend 容器的 `/app/data`，包含：
- 配置文件存储目录（若启用 file backend）
- 未来扩展：SQLite DB、日志等

查看数据卷：
```bash
docker volume inspect agentlens_agentlens-data
```

清理数据：
```bash
docker compose down -v
```

### 反向代理（Nginx / Caddy / Traefik）

前端 80 端口已带 `/api` 反代到 backend。如要直接对外暴露，可只映射 backend 8000 端口，前端走自己的反代。

#### Nginx 外部反代示例

```nginx
server {
    listen 443 ssl;
    server_name agentlens.example.com;

    ssl_certificate     /etc/letsencrypt/live/agentlens.example.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/agentlens.example.com/privkey.pem;

    # 前端
    location / {
        proxy_pass http://127.0.0.1:8080;
    }

    # API 直通（可选，减少一跳）
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header X-Forwarded-For $remote_addr;
    }
}
```

## 方式 B：单独构建镜像

```bash
# Backend
docker build -f Dockerfile.server -t agentlens/server:0.5.0 .

# Frontend
docker build -f Dockerfile.ui -t agentlens/ui:0.5.0 .

# 运行
docker run -d --name agentlens-backend \
    -p 8000:8000 \
    -v agentlens-data:/app/data \
    -e TRACE_STORAGE_BACKEND=file \
    agentlens/server:0.5.0

docker run -d --name agentlens-frontend \
    -p 8080:80 \
    --link agentlens-backend:backend \
    agentlens/ui:0.5.0
```

## 方式 C：传统部署

不用 Docker，直接在主机运行：

```bash
# 后端
cd backend
python -m venv .venv && source .venv/bin/activate
pip install -e ".[dev]"
TRACE_STORAGE_BACKEND=file TRACE_FILE_STORAGE_DIR=./data/traces \
    gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000

# 前端
cd frontend
npm install && npm run build
# 用 nginx / caddy / apache 托管 dist/ 目录
```

### Systemd unit 示例

`/etc/systemd/system/agentlens-server.service`：

```ini
[Unit]
Description=AgentLens Server
After=network.target

[Service]
Type=simple
User=agentlens
WorkingDirectory=/opt/agentlens/backend
Environment="TRACE_STORAGE_BACKEND=file"
Environment="TRACE_FILE_STORAGE_DIR=/opt/agentlens/data"
ExecStart=/opt/agentlens/.venv/bin/gunicorn app.main:app \
    -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

## 升级

```bash
# 拉取新镜像
git pull
docker compose build --pull

# 重启
docker compose up -d
```

数据卷 `agentlens-data` 不会被清除。

## 监控建议

- **日志**：`docker compose logs` 聚合到 ELK / Loki
- **指标**：FastAPI 内置 `/docs` + 推荐加 `prometheus-fastapi-instrumentator` 暴露 `/metrics`
- **健康检查**：`/healthz`（nginx）+ 后端 `python -c "import httpx; ..."`

## 常见问题

### CORS 错误（前端无法访问后端）
修改 `TRACE_CORS_ORIGINS`，加入前端实际来源，例如：
```
TRACE_CORS_ORIGINS='["https://agentlens.example.com"]'
```

### 数据丢失
检查：
1. `TRACE_STORAGE_BACKEND=file` 是否设置（默认是 memory，重启清空）
2. `agentlens-data` 数据卷是否挂载成功：`docker compose exec backend ls /app/data/traces`
3. 容器是否在 `--rm` 模式下运行（默认 compose 不加）

### 前端空白
检查 nginx 配置：`docker compose exec frontend cat /etc/nginx/conf.d/agentlens.conf`
检查后端连通性：`docker compose exec frontend wget -qO- http://backend:8000/api/v1/traces/count/total`
