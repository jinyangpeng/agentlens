# AgentLens · 开发环境启动

## 前置要求

- Python 3.11+
- Node.js 18+ / npm
- Git

## 后端

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate          # Windows
# source .venv/bin/activate      # macOS/Linux
pip install -e ".[dev]"

# 默认内存存储；如需文件存储，复制 .env.example 为 .env 并修改
# TRACE_STORAGE_BACKEND=file
# TRACE_FILE_STORAGE_DIR=./data/traces

uvicorn app.main:app --host :: --port 8000 --reload
```

> **Windows 推荐**：用项目自带的启动脚本，自动处理双栈监听：
> ```bash
> python run.py             # 双栈监听（IPv4 127.0.0.1 + IPv6 ::1），无热重载
> python run.py --reload    # 热重载（IPv6 only，Vite 代理可用）
> ```
> 该脚本设置 `IPV6_V6ONLY=0`，让后端同时接受 IPv4 和 IPv6 连接，
> 解决 Vite 代理走 `::1` 而 uvicorn 默认只绑 IPv4 的 `ECONNREFUSED` 问题。

后端启动在 http://localhost:8000 ，Swagger 文档 http://localhost:8000/docs

## 前端

```bash
cd frontend
npm install
npm run dev
```

前端启动在 http://localhost:5173 ，已配置 `/api` 代理到后端。

## Callback 客户端（AI Agent 端安装）

```bash
cd client
pip install -e .
```

## 验证

1. 后端、前端都启动后，访问 http://localhost:5173
2. 通过 Swagger POST 一个 trace，或运行 AI Agent 接入示例（见 AI-Agent接入说明.md）
3. 在前端列表页查看，点击详情查看调用树/时间线/消息流

## 测试 middleware 追踪

```bash
cd backend
uv run python tests/manual_middleware_test.py
```

刷新 http://localhost:5173，找到名为 `middleware-e2e-test` 的 trace：

- **调用树**：PIIMiddleware / SummarizationMiddleware 节点显示橙色边框 + 齿轮图标 + 类名徽章
- **时间线**：middleware 事件用橙色节点
- **消息流**：在 middleware 触发的消息前显示橙色横幅
- **Header**：显示 `PIIMiddleware` / `SummarizationMiddleware` 徽章
